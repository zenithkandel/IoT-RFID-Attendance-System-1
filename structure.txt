================================================================================
                        PROJECT OVERVIEW
================================================================================

EDU TRACK PRO is a production-ready IoT-based smart attendance management 
system that uses RFID (Radio Frequency Identification) technology to automate
the process of tracking student attendance. The system eliminates manual
attendance marking, reduces errors, and provides real-time data analytics.

================================================================================
                     DATABASE ARCHITECTURE
================================================================================

DATABASE: edutrack (MySQL/MariaDB)
Backend: XAMPP Server (Apache + MySQL)
Server: localhost (default port 80)
Character Set: utf8mb4

TABLE 1: students
─────────────────
  • ID (INT, Primary Key, Auto Increment) - Unique student identifier
  • UID (VARCHAR) - RFID card unique identifier (e.g., "54:83:DE:A4")
  • Name (VARCHAR) - Student full name
  • Roll (INT, UNIQUE) - Roll number (used for login)
  • Class (VARCHAR) - Student's class/grade (e.g., "M-5", "D-7")
  • Address (VARCHAR) - Student address
  • password (VARCHAR) - Bcrypt hashed password for student login
  • last_login (DATETIME) - Last login timestamp

TABLE 2: attendance
───────────────────
  • ID (INT, Primary Key, Auto Increment) - Unique attendance record ID
  • Time (TIME) - Attendance time (HH:MM:SS format)
  • UID (VARCHAR) - Reference to student UID
  • Date (DATE) - Attendance date (YYYY-MM-DD format)
  • Type (ENUM: 'IN', 'OUT') - Check-in or Check-out
  
  Logic: First scan of day = 'IN', Second scan = 'OUT', Third+ = Rejected
  Timezone: Nepal (GMT+5:45) calculated server-side

TABLE 3: users
──────────────
  • ID (INT, Primary Key, Auto Increment) - Unique admin user ID
  • username (VARCHAR, UNIQUE) - Admin username (default: "admin")
  • password (VARCHAR) - Bcrypt hashed password
  • last_login (DATETIME) - Last login timestamp

DATA FLOW:
──────────
  1. Student taps RFID card on reader
  2. ESP8266 reads UID and sends HTTP POST to rfid-checkin.php
  3. PHP validates UID against students table
  4. Checks attendance table for existing scans today
  5. Determines Type: 'IN' (first), 'OUT' (second), or Rejected (third+)
  6. Inserts record with Nepal timezone timestamp
  7. Returns JSON response to ESP8266
  8. ESP8266 displays result on LCD with audio-visual feedback
  9. Web dashboard fetches updated data via AJAX
  10. Real-time UI update without page reload

================================================================================
                        INSTALLATION & SETUP
================================================================================

HARDWARE SETUP:
──────────────
  1. Connect MFRC522 RFID module to ESP8266:
     - SDA  → GPIO 4 (D2)
     - SCK  → GPIO 14 (D5)
     - MOSI → GPIO 13 (D7)
     - MISO → GPIO 12 (D6)
     - RST  → GPIO 5 (D1)
     - GND  → GND
     - 3.3V → 3.3V

  2. Connect LCD I2C to ESP8266:
     - VCC → 5V
     - GND → GND
     - SDA → GPIO 4 (D2)
     - SCL → GPIO 5 (D1)

  3. Connect Buzzer:
     - Positive → GPIO 16 (D0)
     - Negative → GND

  4. Connect LED:
     - Anode (+) → GPIO 15 (D8)
     - Cathode (-) → GND (with 220Ω resistor)

SOFTWARE SETUP:
──────────────
  1. Install XAMPP (Apache + MySQL)
  2. Start Apache and MySQL from XAMPP Control Panel
  3. Import edutrack.sql into phpMyAdmin:
     - Open http://localhost/phpmyadmin
     - Create database 'edutrack'
     - Import edutrack.sql file
  4. Update API endpoint in audrino.ino if needed:
     - serverHost = "192.168.1.100" (your computer's IP)
     - apiEndpoint = "/projects/iot/IoT-RFID-Attendance-System-/API/rfid-checkin.php"
  5. Upload firmware to ESP8266 using Arduino IDE
  6. Configure WiFi:
     - Connect to "AutoConnectAP" hotspot from ESP8266
     - Enter your WiFi credentials
     - Device will auto-connect on next boot
  7. Access dashboards:
     - Admin: http://localhost/projects/iot/IoT-RFID-Attendance-System-/admin-dashboard.html
     - Student: http://localhost/projects/iot/IoT-RFID-Attendance-System-/student-dashboard.html
     - Test: http://localhost/projects/iot/IoT-RFID-Attendance-System-/test-rfid.html

DEFAULT CREDENTIALS:

    Password: admin123 (change immediately after first login)
  
  Students:

TESTING:

================================================================================
                     TROUBLESHOOTING
================================================================================

COMMON ISSUES:
─────────────
  • "Error connecting to database"
    → Check XAMPP MySQL is running
    → Verify database 'edutrack' exists
    → Check conn.php credentials
  
  • "Student not found"
    → Verify UID exists in students table
    → Check UID format matches (e.g., "54:83:DE:A4")
  
  • ESP8266 not connecting to WiFi
    → Reset WiFi settings by holding reset button
    → Reconnect to "AutoConnectAP" and reconfigure
  
  • RFID not reading
    → Check wiring connections
    → Verify MFRC522 module is powered (3.3V)
    → Test with known working RFID card
  
  • LCD not displaying
    → Check I2C address (default: 0x27, some use 0x3F)
    → Update LCD_ADDRESS in firmware if needed
  
  • Timezone issues
    → Nepal timezone (GMT+5:45) is calculated server-side
    → Verify server time is correct
    → Check PHP DateInterval calculation in rfid-checkin.php

LOGS & DEBUGGING:
────────────────
  • Browser Console (F12) - JavaScript errors, API responses
  • Network Tab - API request/response inspection
  • PHP Error Logs - Check XAMPP logs folder
  • Serial Monitor (Arduino IDE) - ESP8266 debug output
